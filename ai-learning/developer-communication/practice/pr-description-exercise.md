# PR Description Exercise

## Goal

Practice writing PR descriptions that reduce reviewer guesswork.

## Scenario

You refactored a pipeline to separate validation from transformation and added tests.

## Tasks

1. Write a rough PR summary.
2. Add why, risk, validation, and review focus.
3. Compare your version with the PR comparison guide.
4. Use AI to reduce repetition and improve clarity.
5. Review whether the final text still exposes the real risk.

## Deliverable

- raw PR description
- structured PR description
- AI-assisted rewrite
- final reviewed version

## Self-Review Rubric

Score your final version from 1 to 5:

- change summary clarity: can the reviewer see what changed quickly
- why clarity: does it explain why the change exists
- risk disclosure: does it expose what could break or what needs attention
- validation quality: does it state how the change was checked
- review guidance: does it tell the reviewer where to focus

## Strong Result Looks Like

- the description explains both behavior and intent
- risk is explicit rather than implied
- tests, validation steps, or manual checks are visible
- the reviewer knows what to verify first
