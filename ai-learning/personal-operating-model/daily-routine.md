# Daily Routine

Use this routine when you want AI to improve consistency and depth, not just short bursts of speed.

## Scenario

You are studying, coding, or designing regularly, but AI usage is becoming uneven.

Some days it helps a lot.

Some days it creates noise.

## Daily Routine Structure

### 1. Define The Main Objective

Write one sentence:

- what are you trying to learn, build, debug, or decide today

Good example:

- understand why schema evolution and schema enforcement diverge in Delta Lake pipeline design

Weak example:

- do some AI learning

### 2. Do The First Thinking Pass Yourself

Before asking AI:

- read the relevant material
- inspect the code, query, or design note
- write down the main uncertainty

### 3. Choose The Smallest Useful Tool Stack

Typical patterns:

- chat assistant for framing
- coding assistant for exact implementation
- research tool for landscape scanning
- note system for durable capture

### 4. Capture One Reusable Insight

At the end of the session, write one short line:

- what changed in your understanding
- what risk became clearer
- what question still remains open

### 5. Convert Output Into Evidence

Do at least one of these:

- run a test
- check documentation
- compare code paths
- restate the design choice in your own words

## Why This Works

This routine forces AI usage to stay attached to a real task, a real uncertainty, and a real validation step.

That keeps speed while protecting understanding.

## Rule

If the day ends with more generated text but no stronger decision, the loop failed.
