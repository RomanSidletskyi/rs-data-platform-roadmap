# AI Practical Exercises

This directory contains exercises where AI is part of the workflow, but never a substitute for the work.

The goal is to practice a disciplined loop:

- inspect the task yourself
- use AI to accelerate reasoning, comparison, or review
- verify the result with your own judgment
- extract reusable lessons from the process

## Exercise Types

- `01_ai_python_refactor/`: use AI to improve structure without losing correctness
- `02_ai_sql_generation/`: use AI to draft and refine SQL while checking semantics closely
- `03_ai_pipeline_design/`: use AI to compare architecture options for data flows and pipelines
- `04_ai_code_review/`: use AI as a review assistant for bugs, regressions, and missing tests
- `05_ai_research_and_tool_comparison/`: compare AI tools by task and defend a selection decision
- `06_ai_technical_writing/`: improve issue notes, summaries, and architecture writing with AI support
- `07_ai_english_for_developers/`: use AI and language tools to improve technical English actively

## Progression

Recommended order:

1. `01_ai_python_refactor/`
2. `02_ai_sql_generation/`
3. `03_ai_pipeline_design/`
4. `04_ai_code_review/`
5. `05_ai_research_and_tool_comparison/`
6. `06_ai_technical_writing/`
7. `07_ai_english_for_developers/`

This order moves from local implementation help toward system thinking, communication, and tool selection.

## How To Work Through Them

1. Try the exercise manually first.
2. Use AI to generate a second pass.
3. Compare assumptions, edge cases, and trade-offs.
4. Keep only the parts that survive technical review.
5. Write down what AI improved and what still required your own judgment.

Several exercises now use a worksheet-style format with scorecards, evidence, reflection prompts, and filled examples.

Treat that pattern as the preferred standard for active practice in this directory.
