# NotebookLM

NotebookLM-style tools are strongest when the task is synthesis across a bounded and deliberate source set.

## What It Is

Use this class of tool after you already gathered the materials and now need structured synthesis, recurring themes, and reusable notes.

## Best For

- studying several docs together
- summarizing a focused source set
- extracting recurring themes, questions, and decisions
- building learning notes from a fixed reading pack

## Not Good For

- open-ended web research without curated sources
- repository-aware coding changes
- replacing first-hand reading of critical material

## Strengths

- good bounded-source synthesis
- useful for learning packs, incident documents, and design-note bundles
- helps convert reading into structured themes

## Weaknesses

- weak if the source pack is poor or incomplete
- can create the illusion of understanding without source reading

## Main Risks

- uploading low-quality material and over-trusting the synthesis
- using it before you know which materials actually matter

## How To Use It Well

- use a constrained source set
- ask for recurring themes, contradictions, trade-offs, and open questions
- convert output into your own notes and decisions

## How Not To Use It

- do not use bounded-source synthesis as a substitute for landscape discovery

## Who Benefits Most

- learners synthesizing several documents
- engineers preparing architecture and research notes from selected sources

## Primary Modes

- learning
- research
- writing

## Example Workflow

- upload notes on Delta Lake, ADLS, and Databricks
- ask for recurring trade-offs around schema evolution, publish boundaries, and recovery design
- convert the output into a decision note or study summary

## Rule

Bounded-source tools are strongest when the source set is deliberate.
