# 01 Analytics Queries

## Overview

These tasks focus on practical SQL analytics patterns used in real data engineering and analytics work.

Suggested dataset tables:

- users
- orders
- order_items
- products
- payments
- events

---

## Task 1 — Revenue Per Customer

### Goal

Calculate total revenue generated by each customer.

### Input

Table:

- orders

Relevant columns:

- order_id
- user_id
- status
- amount
- order_date

### Requirements

- include only completed or paid orders
- group by user_id
- calculate total revenue
- sort by revenue descending

### Expected Output

Columns:

- user_id
- total_revenue

### Extra Challenge

Add:

- order_count
- average_order_value
- first_order_date
- last_order_date

---

## Task 2 — Top 10 Products by Revenue

### Goal

Find the top 10 products that generated the highest revenue.

### Input

Tables:

- order_items
- products

Relevant columns:

- order_items.product_id
- order_items.quantity
- order_items.item_price
- products.product_name

### Requirements

- calculate revenue as quantity * item_price
- aggregate by product_id
- optionally join products for readable names
- sort descending
- return top 10

### Expected Output

Columns:

- product_id
- product_name
- total_revenue

### Extra Challenge

Also return:

- total_quantity_sold
- number_of_orders_containing_product

## Task 5 — Top Customers Per Country

### Goal

Find the top 3 customers by revenue inside each country.

### Input

Tables:

- users
- orders

Relevant columns:

- users.user_id
- users.country
- orders.user_id
- orders.amount
- orders.status

### Requirements

- join users with orders
- include only paid/completed orders
- aggregate revenue per user per country
- use a window function to rank inside each country
- return only top 3

### Expected Output

Columns:

- country
- user_id
- total_revenue
- rank_in_country

### Extra Challenge

Use both:

- ROW_NUMBER
- RANK

and explain the difference.
