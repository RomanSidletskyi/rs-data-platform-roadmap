# Kafka Cookbook Tasks

## Task 1 - Build A Topic Checklist

### Goal

Turn design thinking into a reusable checklist.

### Requirements

- list the questions you would ask before creating a new production topic
- include ownership, key, retention, expected consumers, and replay needs

## Task 2 - Build A Retry And DLQ Recipe

### Goal

Capture a repeatable failure-handling pattern.

### Requirements

- define steps for validation, retry, and DLQ routing
- explain how you distinguish transient and permanent failures

## Task 3 - Build An Operational Triage Recipe

### Goal

Create a small runbook pattern for Kafka incidents.

### Requirements

- define the first 5 signals you would check during a Kafka incident
- explain how those signals help separate broker, consumer, and sink problems
