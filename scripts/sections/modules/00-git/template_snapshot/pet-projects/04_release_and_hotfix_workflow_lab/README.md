# 04 Release And Hotfix Workflow Lab

## Project Goal

Create a guided release and hotfix Git workflow.

## Scenario

You maintain a repository that needs stable releases, tags, and an emergency hotfix path.

## Expected Deliverables

- release branch or tag strategy
- hotfix flow description
- rollback or revert reasoning

## Starter Assets Already Provided

- `.env.example`
- `src/simulate_release_hotfix_flow.sh`
- `data/sample_release_timeline.txt`
- `tests/fixture_expected_hotfix_checklist.txt`

## Definition Of Done

The release and hotfix process is explainable, safe, and consistent with team collaboration.
