# Reference Example ADLS Multi-Engine Contracts And Consumption Lab

This folder is a solved reference example for the guided project:

- `09-azure-data-lake-storage/pet-projects/04_adls_multi_engine_contracts_and_consumption_lab`

## What A Strong Solution Should Demonstrate

- one official publish boundary for supported consumers
- clear separation of internal curated working paths from published interfaces
- explicit schema-change risk reasoning across Databricks, SQL, and BI consumers
- one explanation of how multi-engine consumption raises the need for path governance

## Recommended Contents

- publish contract definition
- engine compatibility notes
- schema review guidance
- examples of supported versus unsupported access paths
