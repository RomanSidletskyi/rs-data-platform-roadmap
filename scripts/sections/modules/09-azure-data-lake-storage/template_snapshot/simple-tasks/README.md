# Azure Data Lake Storage Simple Tasks

These tasks mirror the seven learning blocks in `learning-materials/`.

The purpose is to make the learner explain ADLS storage decisions in small focused exercises before moving into larger labs or projects.

## Task Blocks

1. `01_adls_foundations`
2. `02_namespace_layout_and_storage_modeling`
3. `03_security_identity_and_governance`
4. `04_ingestion_access_and_operating_patterns`
5. `05_cost_performance_and_reliability`
6. `06_adls_in_data_platform_architecture`
7. `07_adls_cookbook`

## Recommended Order

Use tasks immediately after the matching theory block.

The goal is not only to recall Azure storage terminology.

The goal is to force the learner to defend namespace, security, replay, lifecycle, and platform-boundary choices in an ADLS-based data platform.

Each block now also includes one scenario-heavy edge-case task so the learner practices identifying weak storage decisions, not only describing healthy ones.
