# Learning Resources - Architecture Case Studies

## How To Use These Resources

Use case-study resources to practice architectural interpretation, not admiration.

For each case study, extract:

- the core problem pressure
- the 2-3 most important design choices
- one trade-off that smaller teams often miss
- one reason this architecture would be wrong in a simpler environment

## Best Source Types

- engineering blogs
- conference talks
- architecture write-ups from platform teams
- postmortems or design retrospectives when available

## Candidate Companies

- Netflix
- Uber
- Spotify
- Airbnb
- LinkedIn
- Shopify

## What To Look For In Company Write-Ups

- scale and organizational context
- how data moves from source to serving
- what reliability or cost problem forced the design
- which components are foundational and which are company-specific

## Good Reading Rule

Do not ask only "what tools do they use".

Ask:

- what had to be true for this design to make sense
- what simpler design would likely work in a smaller team
