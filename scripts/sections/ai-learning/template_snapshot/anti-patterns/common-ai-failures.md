# Common AI Failures

This document covers the most common ways AI use becomes counterproductive.

## 1. Replacing First Thinking Passes

Bad pattern:

- asking AI before thinking at all

Why it fails:

- the question is weak
- the answer is generic
- understanding stays shallow

Better:

- think first, then ask a narrower question

## 2. Accepting Generated Code Without Review

Why it fails:

- hidden behaviour changes
- missing tests
- incorrect assumptions about local context

Better:

- read the code
- identify what changed
- run validation

## 3. Confusing Fluent Text With Correct Reasoning

Why it fails:

- style can hide incorrect logic
- long explanations often feel more trustworthy than they are

Better:

- ask for constraints, trade-offs, and evidence

## 4. Tool Sprawl

Why it fails:

- too many tools, no stable workflow
- time is lost switching, comparing, and re-learning interfaces

Better:

- pick a small stack by task
- review once a week whether each tool is earning its place

## 5. Translation Instead Of Language Learning

Why it fails:

- you understand the sentence once but do not build skill

Better:

- collect phrases
- practice speaking and writing
- revisit recurring patterns

## 6. Prompt Collecting Instead Of Workflow Design

Why it fails:

- prompts alone do not create repeatable quality

Better:

- design the sequence: think, ask, test, validate, reflect
