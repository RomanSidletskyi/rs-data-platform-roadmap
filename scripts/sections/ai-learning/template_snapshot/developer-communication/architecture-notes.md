# Architecture Notes

Architecture notes should help future readers understand why a design was chosen.

## A Strong Architecture Note Should Include

- problem statement
- constraints
- considered options
- chosen direction
- trade-offs
- operational consequences

## Weak Pattern

- only describing the final architecture
- no rejected alternatives
- no explanation of trade-offs

## Better Pattern

Explain:

- why this option was selected
- what complexity it introduces
- what risks it reduces
- what would change at larger scale or in production

## How AI Helps

Use AI to:

- compare alternatives
- force explicit trade-off language
- rewrite unclear sections

Do not use AI to produce architecture prose before the real constraints are known.
