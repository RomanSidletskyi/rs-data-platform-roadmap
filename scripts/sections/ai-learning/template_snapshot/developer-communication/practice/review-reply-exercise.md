# Review Reply Exercise

## Goal

Practice replying to review comments with clarity, precision, and good engineering tone.

## Scenario

You received a review comment asking why the pipeline now fails earlier for missing required fields.

## Tasks

1. Draft a quick reply.
2. Improve it so it explains the decision and trade-off.
3. Ask AI to make the tone clearer and more professional.
4. Review whether the final version still says something concrete.

## Deliverable

- first reply
- improved reply
- AI-assisted version
- final version

## Self-Review Rubric

Score your final version from 1 to 5:

- directness: does it answer the reviewer question directly
- reasoning quality: does it explain the engineering decision clearly
- trade-off visibility: does it mention the relevant downside or compromise
- tone quality: is it professional without sounding defensive
- concreteness: does it refer to actual behavior, code, or validation instead of vague language

## Strong Result Looks Like

- the response starts with the answer, not filler
- the rationale is short but real
- the message stays respectful under disagreement
- the reviewer can tell whether code or docs need follow-up
