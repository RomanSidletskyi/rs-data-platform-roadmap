# Developer Communication Practice

This directory turns communication guidance into active practice.

The goal is not only to recognize strong patterns.

The goal is to produce them yourself.

## Practice Areas

- `issue-writing-exercise.md`
- `pr-description-exercise.md`
- `adr-exercise.md`
- `review-reply-exercise.md`
- `speaking-about-systems-exercise.md`
- `examples/README.md`

## How To Use

1. Pick one exercise.
2. Write your first version without AI.
3. Compare it with the relevant comparison guide.
4. Use AI to improve structure, clarity, and precision.
5. Produce a final version.
6. Write down what changed and why.

Each exercise now also includes a mini worksheet.

Use it directly inside the file or copy the worksheet into your notes before you start.

## Rule

Use AI to improve communication quality, not to fabricate technical substance.

## Evaluation Rubric

Score each exercise from 1 to 5 on these dimensions:

- clarity: can another engineer understand the message quickly
- completeness: are the needed sections and facts present
- technical honesty: does the text separate facts, assumptions, and open questions
- actionability: can the reader decide what to do next
- conciseness: is the message dense with useful information rather than padded

Use a simple interpretation:

- 1-2: weak, confusing, or missing key information
- 3: acceptable but still creates follow-up questions
- 4: strong and usable in real team work
- 5: clear, precise, and review-ready with minimal ambiguity

## Reflection Prompt

After each exercise, write three short notes:

1. what was weak in the first version
2. what improved in the final version
3. what AI improved versus what you had to fix manually

## Suggested Workflow

Use the worksheet in this order:

1. fill the first-draft sections quickly
2. fill the improved and AI-assisted sections
3. score the final version line by line
4. write one piece of evidence for your lowest score
5. write a short reflection before moving to the next exercise

## Examples Layer

Use the `examples/` directory after you complete your own first attempt.

The examples are not meant to be copied blindly.

They are reference-quality versions that show what strong structure, explicit trade-offs, and technical honesty look like in practice.
