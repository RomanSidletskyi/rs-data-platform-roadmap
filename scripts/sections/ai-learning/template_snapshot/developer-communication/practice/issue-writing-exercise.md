# Issue Writing Exercise

## Goal

Practice turning a vague problem report into a strong engineering issue.

## Scenario

You noticed that a pipeline started failing after an upstream schema change, but the cause is not fully understood yet.

## Tasks

1. Write a weak issue version in one or two lines.
2. Rewrite it into a triage-ready issue.
3. Mark clearly what is known and what is still unknown.
4. Ask AI to improve structure only.
5. Review whether the AI rewrite introduced any unverified claims.

## Deliverable

- weak version
- improved version
- AI-assisted version
- final reviewed version

## Self-Review Rubric

Score your final version from 1 to 5:

- scope clarity: does it identify which pipeline, layer, or workflow is affected
- impact clarity: does it explain what is blocked or degraded
- reproducibility: does it provide enough detail to investigate or reproduce
- uncertainty handling: does it separate known facts from guesses
- triage readiness: can another engineer start working without basic follow-up questions

## Strong Result Looks Like

- the title describes the failure precisely
- the body includes expected versus actual behavior
- impact and scope are explicit
- unknowns are visible instead of hidden
