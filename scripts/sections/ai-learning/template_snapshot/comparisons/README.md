# AI Comparisons

This directory contains decision-oriented comparisons between tools.

The purpose is not to declare one universal winner.

The purpose is to help choose the right tool for the right task.

## Why This Directory Exists

Many AI workflows break because the tool choice is wrong before the work even starts.

Common failure modes:

- using a coding assistant for broad research
- using a chat assistant for file-precise repository edits
- using translation tools as if they were full speaking practice systems
- switching between too many tools without a stable workflow

## Comparison Standard

Good comparisons should explain:

- what task is being evaluated
- which tools are good for it
- which tools are weak for it
- what trade-offs matter most
- when the "best" answer changes because the context changes

Strong comparisons should also rate tools by reusable criteria such as:

- speed
- depth
- context awareness
- reliability
- effort to use well
- cost sensitivity
- privacy sensitivity
- beginner friendliness
- advanced-user leverage

If you already know the task and need a tool fit:

1. `ai-tools-by-task.md`
2. go to the matching file in `../practical-exercises/`

If you already know the language-learning task:

1. `language-tools-by-task.md`
2. continue into `../language-learning/README.md`
3. then use `../practical-exercises/07_ai_english_for_developers/README.md`

If you want a smaller, opinionated working stack by stage:

1. `best-stacks-by-scenario.md`
2. read the matching workflow in `../workflows/`
3. validate the stack in the relevant exercise

## How To Read These Guides

Use `ai-tools-by-task.md` and `language-tools-by-task.md` when you already know the exact task.

Use `best-stacks-by-scenario.md` when you want a smaller, opinionated stack for a broader stage of growth such as beginner learning, active building, architecture work, or English improvement.

Use the richer comparison matrices when you need to justify a choice explicitly rather than only follow a recommended stack.

## Rule

Do not ask "What is the best AI tool?"

Ask "What is the best tool for this exact task, under these constraints?"
