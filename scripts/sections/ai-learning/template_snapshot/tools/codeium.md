# Codeium

Codeium-style assistants are useful as lighter-weight completion and suggestion tools when speed matters more than deep repository-aware analysis.

## What It Is

Use this class of tool for quick drafting once the task is already understood.

## Best For

- fast autocomplete
- quick alternative code snippets
- low-friction drafting of known patterns

## Not Good For

- architecture reasoning
- deep code review
- complex repository-aware refactors

## Strengths

- low friction
- fast completion-oriented workflow
- useful when the engineering decision is already made

## Weaknesses

- shallow context compared with stronger repository-aware assistants
- easy to overuse for problems that need reasoning first

## Main Risks

- substituting completion speed for design judgment
- accepting code without thinking through behavior

## How To Use It Well

- use it after the task is clear
- keep it to localized coding help
- verify output with tests or manual checks

## How Not To Use It

- do not use it as your primary architecture or review tool

## Who Benefits Most

- developers who already know the target pattern and want faster drafting

## Primary Modes

- coding

## Rule

Use it when speed matters more than deep repository-aware analysis.
