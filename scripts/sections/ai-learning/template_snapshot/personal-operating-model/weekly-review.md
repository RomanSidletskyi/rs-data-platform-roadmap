# Weekly Review

Use this review to keep your AI system honest.

## Scenario

You used several tools during the week and made progress, but you need to know:

- what actually helped
- what created noise
- which mistakes are repeating
- which decisions should be written down

## Weekly Review Checklist

### 1. Review Real Outcomes

- what did AI help you finish faster
- where did it save time but reduce clarity
- which outputs survived validation

### 2. Review Tool Quality

For each frequently used tool, ask:

- what exact task did it help this week
- where did it fail
- should it remain in the active stack

### 3. Review Decision Quality

- which technical or workflow decisions were made
- which ones need to be written into the decision journal
- which ones are still unproven

### 4. Review Failure Patterns

- where did you ask overly broad questions
- where did you trust polished text too quickly
- where did you skip validation

### 5. Set The Next Week Constraints

Examples:

- use only one research tool for landscape scans
- always write constraints before architecture prompts
- log one mistake per debugging session

## Why This Works

Weekly review prevents tool sprawl and turns repeated mistakes into explicit system improvements.

## Rule

Do not review how much AI you used.

Review whether the quality of decisions improved.
