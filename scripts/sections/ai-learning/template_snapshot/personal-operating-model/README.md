# Personal Operating Model

This directory turns AI usage from occasional prompting into a repeatable personal system.

The goal is not to use more tools.

The goal is to build habits that preserve understanding, capture decisions, and compound learning over time.

## Why This Layer Exists

Without a personal operating model, AI usage usually becomes one of these patterns:

- random prompting with no retention
- repeated rediscovery of the same ideas
- polished output with weak ownership of the reasoning
- many tools, but no stable working system

This layer is the discipline layer that connects tools, workflows, practice, and reflection.

## What This Directory Covers

- daily routine for focused AI-assisted work
- weekly review for decisions, mistakes, and tool quality
- note capture rules so research and implementation insights do not disappear
- decision journal patterns for architecture and workflow choices
- prompt journal patterns for reusable high-signal prompts
- mistakes log patterns so failure modes become learning assets

## Working Rule

Your system is good if it makes the next correct action easier.

If your notes, prompts, and logs create overhead without better decisions, simplify the system.
