# 02 Team Feature Branch Workflow Lab

## Project Goal

Practice a guided multi-branch collaboration flow.

## Scenario

Simulate a team repository where multiple features move in parallel and must merge back into a stable main branch with reviewable history.

## Expected Deliverables

- branch naming approach
- integration workflow
- merge or rebase policy explanation
- conflict-resolution notes

## Starter Assets Already Provided

- `.env.example`
- `src/simulate_feature_branch_flow.sh`
- `data/sample_branch_map.txt`
- `tests/fixture_expected_review_checklist.txt`

## Definition Of Done

The feature workflow is understandable and repeatable for a small team.
