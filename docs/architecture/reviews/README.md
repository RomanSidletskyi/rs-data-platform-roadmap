# Architecture Review Checklists

This directory contains practical review checklists for evaluating data-platform designs.

Use these after learning the topic basics and before writing an ADR or system design note.

## How To Use These Checklists

Recommended loop:

1. describe the system in plain language
2. choose the closest checklist
3. answer every question with a concrete statement, not a guess
4. mark the highest-risk gaps
5. convert the main decision into an ADR if needed
