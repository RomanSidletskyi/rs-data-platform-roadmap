# Kafka Simple Tasks

These tasks now mirror the same 7-block learning path used in `learning-materials`.

The goal is structural consistency:

- theory lives in one conceptual block
- hands-on practice for that same block lives in the matching task folder

## Task Blocks

1. `01_kafka_foundations`
2. `02_producer_consumer_engineering`
3. `03_event_design_and_schema_governance`
4. `04_kafka_platform_operations`
5. `05_kafka_ecosystem_and_integrations`
6. `06_streaming_architecture_for_data_platforms`
7. `07_kafka_cookbook`

## How To Work Through These Tasks

- read the matching learning-material block first
- complete the practical tasks by hand
- explain the architectural consequence after each exercise
- use `solution.md` as a reference, not as the first step

## Expected Outcome

By the end of these tasks, the learner should be able to connect Kafka mechanics, contract thinking, operations, and system architecture without treating them as separate subjects.
